# ORCHESTRATOR.md - Report Orchestrator 상세 기획서

Report Orchestrator는 이 시스템의 핵심입니다.
**1개 종목 = 1개 오케스트레이터**를 원칙으로, 각 오케스트레이터는 자기 종목의 전체 콘텐츠 제작을 처음부터 끝까지 총괄합니다.

---

## 왜 오케스트레이터 패턴인가?

### 기존 방식의 문제 (단일 플래너)
- 종목 3개를 한번에 기억하고 기획하다 보면 종목 1과 종목 3이 헷갈림
- 기획서가 너무 길어져서 Writer가 놓치는 부분 발생
- 하나에서 에러 나면 전체가 멈춤

### 오케스트레이터 패턴의 장점
| 항목 | 장점 |
|------|------|
| 컨텍스트 명확 | 각 오케스트레이터는 자기 종목만 신경 씀 |
| 진짜 병렬 처리 | 3개 종목 → 3개 오케스트레이터 동시 실행 |
| 에러 격리 | 종목 A 에러가 종목 B, C에 영향 없음 |
| 확장 용이 | 종목 5개? → 오케스트레이터 5개 생성 |

---

## 오케스트레이터 실행 흐름 (1개 종목 기준)

```
Report Orchestrator가 할당받는 정보:
- 종목명 / 티커
- 분석 날짜
- 리포트 출처 (증권사 리포트 URL or 뉴스)
- 콘텐츠 방향 힌트 (선택)

    ↓

Phase 1: 리포트 수집 & 분석 [Stock Analyst Skill 호출]
    - 증권사 리포트 / 공시 / 뉴스 수집
    - 핵심 3가지 포인트 추출
    - 투자자가 알아야 할 숫자 정리
    - 리스크 요인 명시
    → brief.md 생성

    ↓

Phase 2~4: 병렬 실행 ─────────────────────────────┐
    │                                              │
    ▼                                              ▼
[Blog Writer Skill 호출]              [YouTube Script Skill 호출]
    - SEO 최적화 블로그 글 작성              - 후크 (0~15초) 작성
    - H1/H2/H3 구조 설계                   - 본론 구성 (핵심 3가지)
    - 키워드 자연스럽게 배치 (2~3%)         - 차트/데이터 언급 타이밍
    - 공감형 도입 → 데이터 → 전망 → CTA    - 구독/좋아요 CTA
    → blog-post.md 생성                   → youtube-script.md 생성
    │
    ▼
[Image Generator Skill 호출] ← 병렬
    - 실적 비교 차트 이미지
    - 핵심 지표 인포그래픽
    - 산업 구조 다이어그램
    → images/ 폴더에 저장

    ↓

Phase 5: 썸네일 생성 [Thumbnail Creator Skill 호출]
    - 블로그 썸네일 (1200x628)
    - 유튜브 썸네일 (1280x720)
    → thumbnail-blog.png, thumbnail-youtube.png

    ↓

Phase 6: 메타데이터 생성
    - SEO 제목 (60자 이내)
    - 메타 디스크립션 (155자 이내)
    - URL slug (영문, 하이픈)
    - 해시태그 (블로그용 5개, 유튜브용 10개)
    - 발행 권장 시간
    → metadata.json

    ↓

완료 보고 → Content Reviewer로 전달
```

## Phase별 상세 명세

### Phase 1: 리포트 분석 (Stock Analyst Skill)

**인풋:**
- 종목명, 티커, 분석 기준일
- 리포트 출처 (URL or 파일)

**아웃풋 (brief.md):**
```markdown
# [종목명] 분석 기획서 - YYYY-MM-DD

## 종목 기본 정보
- 종목명 / 티커 / 섹터 / 시가총액

## 핵심 포인트 3가지
1. [가장 중요한 투자 포인트]
2. [실적 or 이벤트 포인트]
3. [리스크 or 주의 포인트]

## 주요 수치
- 목표주가: OOO원 (현재: OOO원, 괴리율: OO%)
- PER / PBR / 영업이익률
- 최근 실적 vs 컨센서스

## 리스크 요인
- [리스크 1]
- [리스크 2]

## 콘텐츠 방향
- 블로그 메인 키워드: [키워드]
- 유튜브 후크 아이디어: [후크]
- 타겟 독자: [개인투자자 / 입문자 / 전문투자자]
```

### Phase 2: 블로그 글 작성 (Blog Writer Skill)

**구조 템플릿:**
```
H1: [종목명] 주가 전망 - [핵심 키워드] (날짜)

도입부: 공감형 시작 ("요즘 [종목명] 많이 들으시죠?")
  ↓
H2: [종목명]이 뭐하는 회사야? (입문자용 설명)
  ↓
H2: 이번 리포트 핵심 3가지
  H3: 1. [포인트1]
  H3: 2. [포인트2]
  H3: 3. [포인트3]
  ↓
H2: 숫자로 보는 [종목명]
  - 실적 데이터 표
  - 목표주가 vs 현재가
  ↓
H2: 리스크 - 이것만은 알고 투자하자
  ↓
H2: 내 생각 정리 (필자 관점, 투자 권유 아님)
  ↓
CTA: 유튜브 채널 구독 유도 + 관련 글 내부 링크
  ↓
면책 문구
```

**SEO 규칙:**
- 메인 키워드 밀도: 2~3%
- 글자 수: 1,500~2,500자
- 이미지마다 alt 텍스트 포함
- 내부 링크: 관련 종목 글 연결 (stock-watchlist.md 참조)

---

### Phase 3: 유튜브 스크립트 (YouTube Script Skill)

**구조 템플릿:**
```
[후크 - 0~15초]
"[충격적/호기심 유발 첫 문장]... 오늘 [종목명] 리포트 핵심만 뽑아드립니다."

[본론 - 3~8분 분량]
1. [종목 소개 - 30초]
2. [핵심 포인트 1 - 1분]
3. [핵심 포인트 2 - 1분]
4. [핵심 포인트 3 - 1분]
5. [주요 수치 & 차트 설명 - 1분]
6. [리스크 언급 - 30초]

[클로징 - 30초]
"투자 결정은 여러분의 몫입니다. 도움이 됐다면 구독과 좋아요 부탁드려요."

[면책 문구 자막]
```

**스크립트 규칙:**
- 구어체로 작성 (문어체 X)
- 숫자는 쉽게 ("PER 15배" → "주가가 순이익의 15배")
- [차트 등장] [화면 전환] 등 영상 편집 큐 포함
- 총 스크립트 길이: A4 3~5장 분량

---

### Phase 4: 이미지 생성 (Image Generator Skill)

**생성할 이미지 목록:**
1. **실적 비교 차트**: 최근 4분기 매출/영업이익 바 차트
2. **핵심 지표 카드**: 목표주가, PER, PBR 등 인포그래픽
3. **섹터 내 비교**: 경쟁사 대비 포지셔닝 다이어그램 (선택)

**이미지 스펙:**
- 사이즈: 800x500 (블로그 본문용)
- 스타일: references/image-style-guide.md 참조
- 파일명: `[slug]-chart-01.png`, `[slug]-infographic-01.png`

---

### Phase 5: 썸네일 생성 (Thumbnail Creator Skill)

**블로그 썸네일 (1200x628):**
- 종목명 + 핵심 키워드 텍스트
- 상승/하락 방향 시각적 표현
- 브랜드 컬러 일관성

**유튜브 썸네일 (1280x720):**
- 강렬한 제목 텍스트 (5단어 이내)
- 얼굴 이미지 or 차트 이미지 (선택)
- 클릭률 높은 색상 대비

## 오케스트레이터 병렬 실행 예시

```
사용자: "삼성전자, SK하이닉스, NVIDIA 분석해줘"

Claude Code가 동시에 실행:
┌─────────────────────────────────────────────────────┐
│  Report Orchestrator #1: 삼성전자 (005930)           │
│  Report Orchestrator #2: SK하이닉스 (000660)         │
│  Report Orchestrator #3: NVIDIA (NVDA)              │
└─────────────────────────────────────────────────────┘

각자 독립적으로 Phase 1~6 실행
→ 3개 종목 콘텐츠가 동시에 완성됨

완료 후:
→ Content Reviewer가 3개 글 일괄 검수
→ Outputs/ 폴더에 각각 저장
```

## 에러 처리 원칙

| 상황 | 처리 방법 |
|------|-----------|
| 리포트 URL 접근 불가 | 웹 검색으로 최신 뉴스 대체 수집 |
| 이미지 생성 실패 | 텍스트 기반 대체 이미지 or 스킵 후 보고 |
| 특정 종목 데이터 부족 | 오케스트레이터가 부족함을 명시하고 나머지 진행 |
| 병렬 실행 중 하나 실패 | 해당 종목만 재시도, 나머지는 영향 없음 |

---

## 개선 로드맵

1. **Notion MCP 연동**: Outputs 폴더 결과물을 Notion에 자동 업로드
2. **실적 발표 캘린더 연동**: 구글 캘린더 MCP로 실적 발표일 자동 트리거
3. **댓글 분석 루프**: 유튜브/블로그 댓글 수집 → 다음 콘텐츠 방향 반영
4. **A/B 테스트**: 같은 종목을 다른 앵글로 2가지 버전 제작 후 성과 비교
